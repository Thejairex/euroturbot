# EuroTur — Automatización TourplanNX

## Propósito
Pipeline que lee pagos a proveedores desde Excel, hace login en TourplanNX, busca cada proveedor, navega a Accounting → Transactions, crea un transaction llenando Reference/Voucher/Currency, y sale del proveedor para la siguiente fila.

## Flujo completo (por fila del Excel)

1. **`open_supplier(page, supplier_code)`** — escribe código en `#searchSupplier input[type='text']`, espera `.dropdown table tr`, clickea el que contiene el código, espera botón "Save"
2. **`navigate_to_transactions(page)`** — `page.evaluate` click en `.hamburger` (img SVG), clickea "ACCOUNTING" y "TRANSACTIONS" con `force=True` (por `click-area` overlay)
3. **`INSERT.click()`** — botón en vista Transactions
4. **`create_transaction()`**:
   - Espera que desaparezca `.spinner`
   - REFERENCE: `input.tpdescription-transactionreference` → `{row_index}_{Supplier_Code}_{Voucher_Number}`
   - VOUCHER NO.: `input.tpnumber-vouchernumber` → valor del Excel
   - CURRENCY: usa `page.evaluate()` para encontrar el input con valor "ARS"/"Pesos", setea con `Object.getOwnPropertyDescriptor(...).set`, espera 2s, presiona ArrowDown, clickea `.dropdown table tr` con el código
5. **EXIT modal** → **EXIT proveedor** → vuelve a creditor search

## Datos del Excel

| Columna | Uso |
|---------|-----|
| `Supplier_Code` | Buscar proveedor |
| `Voucher_Number` | VOUCHER NO. en formulario |
| `Service_Cost_Currency` | CURRENCY en formulario |

Headers en fila 2 (0-indexed), datos desde fila 3. Columna A (NaN) se descarta.

## Decisiones técnicas clave

### Selectores
- **Hamburger**: `page.evaluate("document.querySelector('.hamburger').click()")` — no hay role accesible
- **ACCOUNTING / TRANSACTIONS**: `click(force=True)` — `div.click-area` intercepta eventos
- **CURRENCY**: evaluate para cambiar valor nativamente, el input no tiene clase única, se identifica por su valor predefinido
- **Modal dialog**: Playwright `get_by_role("dialog")` funciona (a11y tree) aunque el DOM no tenga `role="dialog"`
- **Dropdown search**: `.dropdown table tr` — estructura de combo Angular con `<table>` interno

### Autenticación
- `.env`: `LOGIN_USERNAME`, `LOGIN_PASSWORD`, `LOGIN_URL`
- Login: fill `input.username`, `input.password`, click `button.tpbutton.login`, espera hash `#/login`

### Tracker
- SQLite con `processed_files` y `processed_rows`
- `--no-tracker` para bypass en desarrollo
- `--tracker status` / `--tracker reset --all`

## Cómo ejecutar

```bash
cd automatizacion
python main.py --test --row 0 --no-tracker                    # modo prueba headless
python main.py --test --row 0 --no-tracker --visible           # modo prueba visible
python main.py --test --no-tracker                             # 1 archivo, 1 fila
python main.py --file "archivo.xlsx" --sheet "HOJA" --no-tracker  # archivo/hoja específico
python main.py --tracker status                                # ver estado del tracker
```

## Issues conocidos

- **Servidor TourplanNX** devuelve 500 intermitente en `GetSessionData` — login falla con `TypeError: Cannot read properties of null`. No es problema nuestro.
- **Cuenta `PPROVEEDORES`** tiene límite de sesiones concurrentes — si alguien más está logueado, no podemos conectar.
- **VOUCHER NO.** se autoformatea con comas (2,149,637) — comportamiento esperado del sistema.
- **CURRENCY dropdown**: solo funciona si el campo tiene valor predefinido ("ARS - Pesos Argentinos" o similar) para identificarlo.
